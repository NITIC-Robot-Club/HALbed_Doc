# DMAを使う理由

たくさんのデータを扱うとき、CPUだけで全部を処理するとほかの仕事に手が回らなくなることがあります。このページでは、DMAを使ってデータのやり取りを任せる方法と、ほかの方法との使い分けを紹介します。

## データ転送を専用回路へ任せる

DMAは、データの運び役をCPUの代わりにしてくれる仕組みです。最初に「どこからどこまで、どれだけ送るか」を決めておくと、CPUが1バイトずつ読み書きしなくても転送できます。

たとえばADCから1000個の測定値を取り込む場合、CPUが毎回データレジスタを読む方法では、そのたびに処理が必要です。DMAを使えば、CPUは転送条件を設定したあと別の仕事を進め、半分または全部の転送が終わった時点で結果を処理できます。

## DMAの利点

- CPUがデータの移動だけに使う時間を減らせる
- UARTやADCなどの連続データを取りこぼしにくくなる
- 一定間隔のデータをまとめて処理できる
- CPUが待ち続ける処理を減らせる

ただし、短い文字列をたまに送るだけなら、DMAの設定や状態管理のほうが複雑になることもあります。

## 3つの方式の使い分け

ポーリングは、完了するまで状態を確認する単純な方式です。処理を理解しやすく、低頻度で短い転送に向いています。

割り込みは、データ到着や転送完了をCPUへ通知します。1バイトずつ割り込みを発生させると、高速通信では割り込み回数が多くなります。

DMAは、まとまった転送や連続データに向いています。転送完了時だけ割り込みを使うことで、CPUの負担を抑えられます。

## DMAを使うときの注意

DMAが動作している間、転送元や転送先のバッファを不用意に書き換えてはいけません。また、ローカル変数を転送バッファにして関数を抜けると、DMA完了前にその領域が無効になる可能性があります。バッファの寿命を十分に長くします。

キャッシュを搭載したSTM32では、CPUが見ているデータとDMAがメモリーへ書いたデータが一致しない場合があります。対象シリーズのキャッシュ管理と、DMAがアクセス可能なメモリー領域を確認してください。

連続受信では、Circular modeやダブルバッファを利用できますが、「DMAが書いている領域」と「CPUが読んでいる領域」を分けて管理する必要があります。

## まとめ

DMAは、大量または連続したデータの移動をCPUから切り離すために使います。高速化だけでなく、CPUがほかの処理を進められることが大きな利点です。一方で、バッファの寿命、同時アクセス、キャッシュを含めた設計が必要です。
