# タイマーとカウンターの基本

マイコンには、時間を数えたり、決まった間隔で処理を動かしたりするためのタイマーがあります。このページでは、タイマーが時間を数える仕組みと、よく使う設定の意味を分かりやすく紹介します。

## タイマーが数えるもの

マイコンのタイマーは、一定の速さで届くクロックを数える部品です。CPUが自分で時間を数え続けなくてもよいため、ほかの処理をしながら時間を測れます。

最初は名前を覚えようとせず、次の4つを「速さ・今の数・終わり・比べる値」と考えると分かりやすくなります。

- `PSC`（Prescaler）: 入力クロックを分周する
- `CNT`（Counter）: 現在のカウント値を保持する
- `ARR`（Auto-Reload Register）: カウントの上限を決める
- `CCR`（Capture/Compare Register）: 比較するカウント値を保持する

## 周期を決める設定

アップカウントでは、一般にカウンターは0からARRまで数えます。タイマー更新イベントの周波数は、概念的には次の式で求められます。

```text
更新周波数 = タイマークロック / ((PSC + 1) × (ARR + 1))
```

たとえばタイマークロックが80 MHzで、PSCを7999、ARRを9999にすると、カウンターは10 kHzで進み、1秒ごとに更新イベントが発生します。

実際の値は、使うタイマーやクロック設定によって変わります。まずはCubeMXに表示される周波数を確認し、必要になったらマニュアルで詳しい条件を調べましょう。

## タイマーでできること

更新イベントで割り込みを起こすと、一定周期の処理を作れます。CCRとCNTを比較して出力を切り替えるとPWMを生成できます。外部信号のエッジでCNTを記録すれば入力の周期を測れます。エンコーダーモードでは、2相の信号から回転方向と移動量を数えられます。

つまり、周期割り込み、PWM、パルス測定、エンコーダー入力は、いずれも「クロックや外部信号を数える」という同じ仕組みを応用した機能です。

## カウンターの桁数

タイマーによってCNTやARRの幅が異なります。16bitなら扱える値は通常0～65535、32bitならより長い周期や細かい分解能を取りやすくなります。必要な周期をARRだけで表せない場合はPSCも使って調整します。

## まとめ

タイマーは、クロックを分周して数える回路です。PSCが数える速さ、ARRが周期、CCRが比較位置を決めると考えると、TickerやPWMの設定も理解しやすくなります。
