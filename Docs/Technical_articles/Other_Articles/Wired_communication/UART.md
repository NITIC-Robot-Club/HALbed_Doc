# UART / USART / LPUARTの違いについて

UART, USART, LPUARTが並んでいると、なにが違うのか、使っていいのかと迷うと思います。

一言でいえば、USARTやLPUARTも、TX/RXを使った普通のUART通信として使えます。
ただし、名前が違うのは、それぞれ追加機能や得意な用途が違うためです。

この資料では、UARTを基準にして、USARTとLPUARTで何が追加されているのかを整理します。

## UARTってなに？

はじめに、UARTについて簡単に振り返ってみましょう。
UARTは、非同期式のシリアル通信を行うための回路です。

クロック線を使わず、`TX` と `RX` の2本でデータを送受信します。
送信側と受信側は、あらかじめ同じ通信速度に設定しておき、そのタイミングを基準にしてデータを読み取ります。

UARTは、次のような用途でよく使われます。

- PCとのデバッグ通信
- マイコン同士の通信
- 無線モジュールとの通信
- センサやモータドライバとの通信

UARTで重要なのは、通信相手と次の設定をそろえることです。
```text
通信速度
データ長
パリティ
ストップビット
信号レベル
```

この設定がずれていると、文字化けしたり、受信できなかったりします。

## USARTってなに？

次に、UARTの中にSが入ったUSARTについてです。
USARTは、UARTの非同期通信に加えて、同期式通信にも対応できるペリフェラルです。

名前は次の意味です。

```text
UART  : Universal Asynchronous Receiver Transmitter
USART : Universal Synchronous / Asynchronous Receiver Transmitter
```

`Asynchronous` は非同期、`Synchronous` は同期という意味です。

UARTでは、通信にクロック線を使いません。
一方、USARTの同期式通信では、データ線とは別にクロック線を使って通信できます。

STM32では、USARTを有効にすると、次のようなピンが出てくることがあります。

```text
TX : 送信データ
RX : 受信データ
CK : 同期クロック
```

`CK` は、同期式通信で使うクロック出力です。
非同期UARTとして使う場合、基本的に `CK` は使いません。

USARTの特徴は、UARTよりも対応できる通信モードが多いことです。
STM32のUSARTでは、機種によって次のような機能を持つ場合があります。

```text
非同期通信
同期通信
LIN
IrDA
SmartCard
Half-Duplex
Hardware Flow Control
Multi-Processor Communication
```

各シリーズで対応している機能は異なります。
使いたい機能が対応しているかなどは、リファレンスマニュアルを確認してください。

USARTには、様々な機能があります。下の資料はG4シリーズのUSART機能を紹介している資料です。
- [STM32G4-USART](https://www.stmcu.jp/wp/wp-content/uploads/files/presentation-ja/STM32G4/32_STM32G4-Peripheral-USART-interface-(USART)-J-.pdf) : G4シリーズのUSART機能紹介資料


普段よく使うのは非同期通信です。
そのため、CubeMXで `USART1` と表示されていても、多くの場合は `Asynchronous` に設定して、TX/RXだけを使います。これで、UARTと同じように使えます。


### USARTのHalf-Duplex

Half-Duplexは、送信と受信を同じ1本の信号線で行うモードです。

通常のUARTでは、送信用の `TX` と受信用の `RX` を分けます。

```text
通常のUART
TX ----> RX
RX <---- TX
```

Half-Duplexでは、1本の線を送信にも受信にも使います。

```text
Half-Duplex
TX/RX <----> TX/RX
```

配線を減らせる一方で、同時に送信と受信はできません。
また、送信状態と受信状態の切り替えを意識する必要があります。

サーボモーターなどでは、Half-Duplexで通信するものがあります。

## LPUARTってなに？

LPUARTは、Low Power UARTの略です。
低消費電力での動作を意識したUARTです。

通常のUART/USARTと大きく違う点は、低消費電力モード中の動作や復帰に向いていることです。

バッテリー駆動の機器では、マイコンを常に動かし続けると消費電力が大きくなります。
そのため、普段はSTOPモードなどの低消費電力状態に入り、必要なときだけ起動する構成にします。

LPUARTは、このような場面で使いやすいように作られています。

```text
低消費電力モード中に通信を待つ
UART受信をきっかけにマイコンを起こす
低速クロックでシリアル通信を行う
バッテリー駆動機器で通信する
```

### LPUARTとクロック

通常のUSARTでは、比較的高い周辺クロックを使ってボーレートを作ります。
一方でLPUARTは、低速なクロック源でもUART通信ができるように設計されています。

例えば、低消費電力モード中でも動作できるクロックを使い、受信待ちを続けるような使い方ができます。

ただし、使えるクロック源や低消費電力モード中の動作可否は、STM32のシリーズや設定によって変わります。
実際に使うときは、対象マイコンのリファレンスマニュアルで確認する必要があります。

### LPUARTを使うときの注意点

LPUARTは低消費電力用途に向いていますが、万能ではありません。

通常のUSARTと比べて、次のような制限がある場合があります。

```text
最大通信速度が低い
使える機能が少ない
対応しているピンが限られる
低消費電力モード中に使える条件がある
```

特に、高速通信をしたい場合や、LIN、SmartCard、同期通信などの機能を使いたい場合は、USARTを選ぶ方が分かりやすいです。

LPUARTは、「低消費電力でUART通信を待ちたいときに使うもの」と考えるとよいです。


## 比較表とまとめ

| 種類 | 主な目的 | 特徴 | 向いている用途 |
|---|---|---|---|
| UART | 基本的な非同期シリアル通信 | TX/RXで通信する基本形 | デバッグ通信、簡単なマイコン間通信 |
| USART | 多機能なシリアル通信 | 非同期通信に加えて同期通信などにも対応 | 通常のUART通信、高速通信、追加機能を使う通信 |
| LPUART | 低消費電力向けUART | 低速クロックや低消費電力モードと相性がよい | バッテリー駆動、STOPモードからの復帰 |

USARTやLPUARTという名前に戸惑うことがありますが、どの機能を使うかを見れば整理できます。
PCやセンサーとの比較的低速で一般的な通信であれば、どれを使っても大きな違いはありません。
ほとんど、これらの違いを意識せずに、UART通信として使えます。

それぞれに特徴があるので、必要に応じて使い分けましょう！
