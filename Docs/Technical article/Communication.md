# 通信プロトコル基礎（UART / I2C / SPI / CAN / USB 等）

## 共通の基礎概念
- 物理層と論理層: 電圧レベル、差動/単端、フレーミング、ビット順
- 同期方式: 非同期（UART）と同期（SPI/I2C/CAN）
- タイミング: ボーレート、クロックジッタ、サンプリングポイント
- エラーチェック: パリティ、CRC
- フロー制御: ハードウェア（RTS/CTS）/ソフトウェア（XON/XOFF）

## UART（シリアル）
- 非同期通信。スタートビット・データビット・パリティ・ストップビットで構成。
- ボーレート精度はクロックに依存。
- 用途: デバッグログ、簡易デバイス間通信。

## I2C
- マスター/スレーブ構成、7/10 ビットアドレス。
- バスはオープンコレクタ（プルアップ必須）。クロックストレッチに対応する必要あり。
- 複数デバイスのアドレス衝突に注意。

## SPI
- フルデュプレックス、マスターがクロック供給。
- 設定: クロック極性（CPOL）/クロック位相（CPHA）、ビット順、チップセレクト管理。
- 高速・短距離のデータ転送向け。

## CAN
- フレームベースのメッセージング、識別子による優先度付け。
- 物理層は差動（CAN_H/CAN_L）、バス終端（120Ω）が必要。
- エラーハンドリングとビットモニタリングを活用する。

## USB / Ethernet（簡潔）
- USB: ホスト/デバイス、クラス（CDC、HID 等）。PHY とコントローラ両方を理解する。
- Ethernet: PHY, MAC, フレーム、MTU、コリジョン管理（古い規格）

## 実装上のヒント
- まず低速・余裕のあるタイミングで動作確認を行う。
- ログ（UART）を確保してデバッグしやすくする。
- 割り込みは短く、重い処理はタスク/スレッドやDMAへ移す。
- ハードウェアレベル（終端、プルアップ、グラウンド）を優先して検証する。
