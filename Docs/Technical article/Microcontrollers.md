# マイコン基礎

## 概要
マイコン（MCU）はCPU、メモリ、周辺機能（GPIO / Timer / ADC / DMA 等）を1チップに集積した組込み用コンピュータです。組込み開発ではリソース制約、電力、リアルタイム性を常に意識します。

## 主な構成要素
- コア: ARM Cortex-M 系など
- クロック・電源: クロックツリー、PLL、低消費電力モード
- メモリ: フラッシュ（コード）、SRAM（データ）
- 周辺: GPIO, UART, I2C, SPI, ADC, DAC, Timers, DMA, CAN, USB
- 割り込み: NVIC 等の割り込みコントローラ

## 開発ワークフロー（短縮）
1. ピン・クロック・ペリフェラルの設計（CubeMX 等ツール使用が一般的）
2. HAL/LL 等の初期化コード生成
3. 割り込みとDMAの設計（リアルタイム性を確保）
4. ビルド→フラッシュ→デバッグ（SWD/JTAG）

## よくある落とし穴と対策
- クロック設定ミス: ボーレートやタイミングが狂う。
- ピン多重機能（AF）忘れ: ペリフェラルが動かない。
- 割り込み優先度の誤設定: デッドライン逸脱や優先度逆転。
- 電源/グラウンド配慮不足: ADCなどでノイズが増える。

## 確認チェックリスト（開発前）
- クロックソースと周波数は適切か
- ピン設定（入出力/AF/プルアップ）は正しいか
- 電源のデカップリングはされているか
- 割り込み優先度とスタックサイズは十分か
